# テンプレート作成ガイド

このシステムは、Google スライドの「テーマ（マスター）」に定義されたレイアウトとプレースホルダを利用してスライドを生成します。
独自のテンプレートを使用する場合、以下のルールに従ってレイアウト名とプレースホルダを設定することで、綺麗にコンテンツを流し込むことができます。

## 1. レイアウト名の対応表

システムは、生成するスライドの種類（Type）に応じて、以下の優先順位でレイアウトを検索・適用します。

| スライド種類 (Type) | 対応するレイアウト表示名 (推奨) | 代替レイアウト (自動フォールバック) |
| :--- | :--- | :--- |
| **Title** (表紙) | `TITLE` | なし (必須) |
| **Section** (章区切り) | `SECTION`<br>`SECTION_HEADER` | `SECTION ONLY` 等 |
| **Content** (箇条書き) | `CONTENT`<br>`TITLE_AND_BODY` | - |
| **Agenda** (目次) | `AGENDA` | `CONTENT` or `TITLE_AND_BODY` |
| **Conclusion** (まとめ) | `CONCLUSION` | `CONTENT` or `TITLE_AND_BODY` |
| **Diagram** (図解全般)* | `TIMELINE`, `PROCESS` 等 (あれば) | **`CONTENT`**<br>(特定のレイアウトが無い場合、自動的に `CONTENT` が使用されます) |

※ `AGENDA` や `CONTENT` などの名前は、マスタースライドの「名前の変更」で設定された**表示名 (Display Name)** と一致している必要があります。
※ 図解用レイアウト（TIMELINE等）を個別に用意しない場合でも、`CONTENT` レイアウトがあればヘッダー・フッター付きのスライドとして生成されます。

> (*) **Diagram** には以下のタイプが含まれます:
> `timeline`, `process`, `cycle`, `pyramid`, `triangle`, `compare`, `statsCompare`, `barCompare`, `cards`, `headerCards`, `kpi`, `table`, `faq`, `progress`, `quote`, `imageText`, `stepUp`, `flowchart`, `diagram`

## 2. プレースホルダの設定 (重要)

スライド内のコンテンツ配置は、マスタースライド上の「プレースホルダ」によって制御されます。

### 基本ルール
*   **タイトル**: `TITLE` または `CENTERED_TITLE` タイプのプレースホルダに流し込まれます。
*   **サブタイトル**: `SUBTITLE` タイプのプレースホルダに流し込まれます。

### 図解 (Diagram) の配置エリア
図解タイプのスライド（タイムライン、KPIカードなど）では、プログラムが以下の優先順位でプレースホルダを探し、その**位置とサイズに合わせて**図解を描画・配置します。

1.  **Body (本文)**: 標準的なテキスト用プレースホルダ
2.  **Object (オブジェクト)**: 図形やグラフ用の汎用プレースホルダ
3.  **Picture (画像)**: 画像流し込み用のプレースホルダ

**テンプレート作成のコツ:**
*   図解を表示させたい位置に、上記のいずれかのプレースホルダを配置してください。
*   例えば「左にテキスト、右に図解」というレイアウトを作りたい場合、右側に「画像プレースホルダ」を置いておけば、そこに KPI カードやグラフが綺麗に収まります。
*   生成された図解は、プレースホルダの**中央 (Center/Middle)** に配置されます。

### プレースホルダが見つからない場合
適切なプレースホルダが見つからない場合、システムはデフォルトの座標（スライド中央付近）に強制的に描画しようとします。レイアウト崩れを防ぐため、可能な限り `TITLE_AND_BODY` （タイトルと本文）レイアウトを用意するか、カスタムレイアウトに上記のプレースホルダを含めることを推奨します。
